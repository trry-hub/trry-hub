解决 mpvue 自带的语法检测的规则与 vscode 自带的格式化规则冲突的问题。

eslint 语法检测

mpvue 使用 vue-cli 脚手架搭建项目，默认使用 eslint 语法检测，不符合规则的代码将会报错无法运行。

  • 代码末尾不能加分号 ;
  • 代码中不能存在多行空行
  • tab 键不能使用，必须换成两个空格
  • 代码中不能存在声明了但未使用的变量

语法检测的自定义规则在项目根目录里的 .eslintrc.js 的 rules 中配置。

json
/*
* "off" 或者 0    //关闭规则
* "warn" 或者 1    //作为警告
* "error" 或者 2    //作为一个错误
*/
'rules': {
    // allow paren-less arrow functions
    'arrow-parens': 0,
    // allow async-await
    'generator-star-spacing': 0,
    // allow debugger during development
    'no-debugger': process.env.NODE_ENV === 'production' ? 2 : 0,
    "no-unused-vars": [2, { 
      // 允许声明未使用变量
      "vars": "local",
      // 参数不检查
      "args": "none" 
    }],
    // 关闭函数括号前的空格验证
    'space-before-function-paren': 0
  }

vscode 格式化规则

vscode 默认使用 Vetur + Prettier 格式化代码，快捷键可以在 Keyboard Shortcuts 中设置。

Keyboard

Format

vetur 默认格式化代码为双引号、代码结尾带分号，eslint 直接报错。

解决这个问题需要先安装 Prettier 插件

image.png

然后就可以在 User Settings 中新增自定义的规则：

json
{
  "files.autoSave": "onFocusChange",//窗口失去焦点时自动保存
  "prettier.singleQuote": true,     //使用单引号
  "prettier.semi":false,            //不加分号
  "javascript.format.insertSpaceBeforeFunctionParenthesis": true //
}
